MANUAL PUBLISH

本地发布流程

  1. 在线后台保存内容后，源码已经进入 Hyper001-source
  2. 回到本地项目目录 C:\Users\31847\Documents\Blog\Hyper001
  3. 执行 npm run publish:manual
  4. 等待命令完成，前台会被部署到 Hexo.github.io
推荐做法

每次写完一篇再统一发布即可，不需要每改一处就发布一次。

CURRENT MODE

为什么现在是手动发布

  • 在线后台负责编辑和保存源码，不再依赖当前失效的 GitHub Actions 自动触发。
  • 这样你仍然能在线写作，同时避免每次保存都遇到自动工作流失败。
  • 等后面条件允许，我再帮你把自动发布链路重新接回来。
